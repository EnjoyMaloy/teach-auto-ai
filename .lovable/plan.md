

# Фикс консистентности персонажа-маскота

## Проблема

Сейчас маскот описывается только текстом. Каждый вызов генерации изображений интерпретирует описание по-своему — отсюда разный вид персонажа на каждой картинке.

## Решение

Генерировать **референсное изображение маскота** на первом шаге, а затем передавать его как визуальный референс в каждый последующий запрос генерации картинок.

## Технический план

### 1. Генерация референс-изображения маскота

В `useGenerateCourse.ts` — после получения текстового описания маскота, делаем дополнительный вызов `generate-image` с промптом "нарисуй персонажа на белом фоне в полный рост". Полученное base64-изображение сохраняем в storage bucket `mascots` и передаём URL дальше.

### 2. Обновление edge function `generate-image`

Добавить поддержку параметра `referenceImageUrl`. Если он передан — включать изображение как `inline_data` часть в запрос к Gemini API (multimodal input: текст + изображение). Промпт будет содержать инструкцию: "The attached image shows the exact character that MUST appear in this illustration. Copy this character exactly — same face, proportions, colors, outfit."

### 3. Передача референса в каждый запрос

В `generateImageForSlide` добавить параметр `mascotImageUrl` и прокидывать его через весь flow генерации (обычная генерация и MD-импорт).

```text
Flow:
  1. AI описывает маскота (текст) ← уже есть
  2. NEW: Генерируем картинку маскота → загружаем в storage → получаем URL
  3. Каждый generate-image получает referenceImageUrl + текстовое описание
  4. Gemini видит реальное изображение персонажа → копирует его точнее
```

### Изменяемые файлы

- `supabase/functions/generate-image/index.ts` — multimodal input с референсом
- `src/hooks/useGenerateCourse.ts` — генерация и сохранение референс-изображения, передача URL
