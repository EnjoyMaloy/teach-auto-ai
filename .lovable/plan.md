

# UX/UI Аудит: Раздел Дизайн-системы

## Общая оценка

Текущий редактор дизайн-системы функционально богат, но имеет ряд проблем с юзабилити и визуальной организацией. Ниже представлен детальный анализ и рекомендации по улучшению.

---

## 1. Проблемы информационной архитектуры

### Текущее состояние
Вкладки организованы в сетку 3x2 с нелогичным порядком:
```
Фон и кнопки | Интерактив | Блоки
Шрифты      | Звуки      | Маскот
```

### Проблемы
- **Перегруженность вкладки "Фон и кнопки"**: содержит 3 крупные секции (фоны, акцентный цвет, кнопки) — слишком много для одной вкладки
- **Вкладка "Интерактив"** содержит разнородный контент: цвета ответов + подсказки + Rive-маскот (маскот дублируется на двух вкладках)
- **Названия вкладок** не всегда понятны: "Блоки" — что именно там настраивается?

### Рекомендация
Реорганизовать вкладки по логическим группам:

```
Тема      | Кнопки    | Текст
Квизы     | Звуки     | Маскот
```

Или использовать вертикальную навигацию для лучшей читаемости на узкой панели 420px.

---

## 2. Проблемы визуальной иерархии

### Текущее состояние
- Все секции выглядят одинаково: Label + описание + контрол
- Нет визуального разделения между крупными блоками настроек
- Только `border-t pt-6` используется как разделитель

### Проблемы
- Сложно сканировать глазами длинные формы
- Пользователь теряется в потоке однотипных полей
- Нет "якорей" для быстрой навигации

### Рекомендации
1. **Группировать настройки в карточки** с заголовками:
   ```
   ┌─────────────────────────────────┐
   │ 🎨 Фоны курса                   │
   │   [Сетка фонов]                 │
   └─────────────────────────────────┘
   
   ┌─────────────────────────────────┐
   │ 🔘 Кнопки                       │
   │   [Цвет] [Форма] [Объём]        │
   └─────────────────────────────────┘
   ```

2. **Использовать Collapsible-секции** для продвинутых настроек
3. **Добавить визуальные разделители** между логическими группами

---

## 3. Проблемы с превью

### Текущее состояние
- Превью кнопки находится внизу секции кнопок
- Превью прогресс-бара и выбора ответа — в секции акцентного цвета
- Превью шрифтов — под каждым селектором

### Проблемы
- **Разрозненные превью** не дают целостной картины
- Пользователь вынужден прокручивать туда-сюда между настройками и превью
- Нет контекста: как кнопка смотрится НА фоне курса?

### Рекомендации
1. **Sticky-превью**: закрепить мини-превью в правой панели при прокрутке
2. **Контекстные превью**: показывать элемент в реальном контексте (кнопка на фоне курса, а не на сером)
3. **Live-обновление**: текущее превью в `DesignPreviewBlocks` уже хорошо работает, но оно слишком далеко от настроек

---

## 4. Проблемы юзабилити Color Picker

### Текущее состояние
```
[Цветной квадрат] [#FFFFFF]
```

### Проблемы
- Маленький размер цветового квадрата (40x40px)
- Нет возможности копировать/вставлять HEX
- Нет пресетов для часто используемых цветов
- Нет цветовой пипетки (eyedropper)

### Рекомендации
1. **Увеличить область клика** цветового квадрата до 48-56px
2. **Добавить кнопку копирования** HEX-кода
3. **Добавить цветовую палитру** из выбранной темы для быстрого доступа
4. **Показывать контрастность** для пар цветов (кнопка/текст на кнопке)

---

## 5. Проблемы с выбором темы

### Текущее состояние
- Темы отображаются в виде карточек 2 колонки
- Превью темы — 3 цветных кружка
- Кнопки редактирования появляются при hover

### Проблемы
- **Превью не информативно**: 3 кружка не показывают, как выглядит тема
- **Кнопки на hover** могут быть не обнаружены на мобильных устройствах
- **Нет визуального отличия** между "Общие темы" и "Мои темы"

### Рекомендации
1. **Улучшить превью темы**: показать мини-карточку с кнопкой и текстом
   ```
   ┌─────────────────┐
   │  ■ Заголовок    │
   │  Текст          │
   │  [Кнопка]       │
   │─────────────────│
   │  Название темы  │
   └─────────────────┘
   ```

2. **Добавить контекстное меню** (три точки) для действий вместо hover-иконок
3. **Визуально разделить** общие и личные темы (разный фон секций, иконки)

---

## 6. Проблемы с фоновыми пресетами

### Текущее состояние
- Сетка 3 колонки, карточки 80px высотой
- Кнопка "По умолч." появляется внизу при hover
- Иконки Edit/Delete появляются вверху при hover

### Проблемы
- **Слишком маленькие карточки** для оценки градиента
- **Дублирование информации**: название внутри карточки + "по умолчанию" badge
- **Неочевидная механика**: клик = выбрать для просмотра, "По умолч." = применить ко всем — это путает

### Рекомендации
1. **Увеличить размер карточек** или показывать 2 в ряд
2. **Упростить механику**: один клик = выбрать как дефолт, shift+клик = только превью
3. **Добавить drag-and-drop** для изменения порядка
4. **Показать угол градиента** визуально (стрелка или число)

---

## 7. Проблемы с вкладкой Маскот

### Текущее состояние
- Длинная форма с множеством полей
- Rive-настройки показываются только если включены
- Форма занимает несколько экранов прокрутки

### Проблемы
- **Слишком много полей** для начального этапа
- **Нет визуальной подсказки** какие поля обязательны
- **Сложный Rive-воркфлоу**: пользователь должен знать названия state machine и inputs

### Рекомендации
1. **Разделить на шаги** (wizard):
   - Шаг 1: Базовое описание (имя, промпт)
   - Шаг 2: Визуальный стиль
   - Шаг 3: Rive-анимация (опционально)

2. **Автоопределение Rive-параметров**: при загрузке .riv файла парсить state machines и inputs
3. **Добавить превью маскота** в реальном контексте (на слайде квиза)

---

## 8. Проблемы доступности

### Выявленные проблемы
- Некоторые интерактивные элементы не имеют `aria-label`
- Color picker недоступен для пользователей с нарушениями зрения
- Hover-only элементы недоступны с клавиатуры

### Рекомендации
1. Добавить `aria-label` ко всем иконкам-кнопкам
2. Показывать название цвета или HEX рядом с пикером
3. Заменить hover-only элементы на always-visible или keyboard-accessible

---

## 9. Производительность и отзывчивость

### Текущее состояние
- Debounce 800ms для auto-save
- Мгновенное обновление превью

### Проблемы
- **Нет индикации сохранения**: пользователь не знает, сохранены ли изменения
- **Нет отмены изменений**: случайно изменил — нельзя откатить

### Рекомендации
1. **Добавить индикатор статуса**: "Сохранено" / "Сохранение..." / "Ошибка"
2. **Добавить кнопку Reset**: сброс к дефолту темы
3. **Добавить Undo/Redo** для критичных изменений

## Резюме

Основные направления улучшения:
1. **Упростить информационную архитектуру** — логичнее группировать настройки
2. **Усилить визуальную иерархию** — карточки, разделители, collapsible-секции  
3. **Улучшить превью тем** — показывать реальный внешний вид, а не абстрактные кружки
4. **Добавить обратную связь** — индикаторы сохранения, статусы
5. **Увеличить touch-target** для мобильных пользователей
