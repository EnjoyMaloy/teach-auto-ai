
# Единая история чата в ИИ-сайдбаре

## Что сейчас

Сайдбар имеет 3 отдельных потока сообщений, которые не связаны между собой:
1. **Генерация курса** -- рендерится из `useAIGeneration`, пропадает при смене режима
2. **Редактирование блока** -- показывает спиннер, но не сохраняет историю
3. **Свободный чат** -- хранит `chatMessages[]`, но только в idle-режиме

При переключении между режимами история теряется -- пользователь не видит свои предыдущие действия.

## Что будет

Единый непрерывный чат, где все действия идут по порядку (как в Lovable):

```text
[ Промпт пользователя на создание курса     ]  <- user
[ Блок прогресса генерации (шаги, полоски)   ]  <- generation
[ Курс создан! 5 уроков, 42 сек             ]  <- completion
[ Измени текст в блоке 2                    ]  <- user
[ Готово! Обновлено 3 элемента              ]  <- assistant
[ Добавь ещё один урок                      ]  <- user
[ ...                                        ]  <- assistant
```

Переключение режимов (Создать курс / Ред. блок / idle) **не очищает** историю -- меняется только панель настроек и поведение ввода.

## Технические детали

### Файл: `src/components/editor/EditorAISidebar.tsx`

**1. Новый тип сообщений**

Заменяем `ChatMessage` и отдельный рендеринг генерации на единый массив:

```typescript
interface UnifiedMessage {
  id: string;
  type: 'user' | 'assistant' | 'generation' | 'completion' | 'error';
  content: string;
  timestamp: number;
  steps?: GenerationStep[];     // для type === 'generation'
  isGenerating?: boolean;       // для type === 'generation'
  lessonCount?: number;         // для type === 'completion'
  duration?: number;            // для type === 'completion'
}
```

Состояние: `const [messages, setMessages] = useState<UnifiedMessage[]>([])` -- заменяет `chatMessages`.

**2. Изменения в handleSubmit**

- **Режим generate**: добавляем `user` сообщение с промптом, затем `generation` сообщение (пустые шаги). Запускаем `runGeneration`.
- **Режим edit-block**: добавляем `user` сообщение, вызываем API, добавляем `assistant` с результатом.
- **Режим idle**: то же самое -- `user`, API, `assistant`.

**3. Синхронизация прогресса генерации**

Добавляем `useEffect`, который следит за `state.steps` и `state.status` из `useAIGeneration`:
- Обновляет последнее `generation` сообщение в массиве (его `steps` и `isGenerating`)
- При `status === 'completed'` -- добавляет `completion` сообщение (с защитой от дублей через ref)
- При `status === 'error'` -- добавляет `error` сообщение

**4. Единый рендеринг**

Убираем все условные блоки (`mode === 'idle' && chatMessages...`, `isGenerating || isCompleted`, `isError`). Вместо них -- один `messages.map()`:

- `user` -- пузырь справа (как сейчас)
- `assistant` -- пузырь слева (как сейчас)
- `generation` -- карточка с шагами и прогресс-барами слева (существующий UI генерации)
- `completion` -- зелёная карточка "Курс создан!"
- `error` -- красная карточка с кнопкой "Попробовать снова"

**5. Пустое состояние**

Маскот показывается только когда `messages.length === 0`, независимо от режима.

**6. Панель настроек генерации**

Блок с выбором дизайн-системы, иллюстраций и кол-ва уроков по-прежнему показывается когда `mode === 'generate'` и нет активной генерации. Он отображается **над** чатом (между шапкой и областью сообщений).

- `useAIGeneration.tsx` -- без изменений
- `useGenerateCourse.ts` -- без изменений
- Edge-функции -- без изменений
- `Editor.tsx` -- без изменений
