

## Миниатюры блоков в таймлайне: настоящий масштабированный превью

### Проблема
Сейчас миниатюры в таймлайне рендерятся вручную (показывается только картинка или иконка типа). Это выглядит неинформативно -- не видно реальное содержимое блока.

### Решение: CSS transform scale
Рендерить настоящий `MobilePreviewFrame` в полном размере (390x760), но внутри контейнера 72x128 с CSS `transform: scale(~0.17)`. Браузер отрисует полноценный блок, а потом уменьшит визуально. Получится точная миниатюра, как в скриншоте.

```text
+------------------+       transform: scale(0.17)       +------+
|  390 x 760       |  =============================>    | 72px |
|  Full block      |                                    | 128px|
|  with all        |                                    | mini |
|  content         |                                    +------+
+------------------+
```

### Что будет сделано

**1. Новый компонент `ScaledBlockThumbnail`** (внутри `CourseTimeline.tsx`)

- Контейнер 72x128 с `overflow: hidden`
- Внутри: div 390x760 с `transform: scale(0.168)` и `transform-origin: top left`
- Внутри div: `MobilePreviewFrame` с `isReadOnly=true`, `embedded=true`, `fillContainer=true`
- `pointer-events: none` -- клики обрабатываются контейнером, не превью
- Бейдж с номером блока и иконкой типа поверх миниатюры (как сейчас)

**2. Обновление `CourseTimeline.tsx`**

- Заменить `MiniBlockPreview` на `ScaledBlockThumbnail`
- Передать `designSystem` в пропсы `CourseTimeline` (нужно добавить)
- Каждый блок рендерится через реальный `MobilePreviewFrame`

**3. Пропс `designSystem` в `CourseTimeline`**

- Добавить `designSystem?: CourseDesignSystem` в интерфейс `CourseTimelineProps`
- Пробросить из родительского компонента (Editor)

### Технические детали

- Масштаб: 72/390 = ~0.185 по ширине, 128/760 = ~0.168 по высоте. Берем min = 0.168
- `MobilePreviewFrame` уже поддерживает `fillContainer` и `isReadOnly` -- не нужно менять
- `pointer-events: none` на превью предотвращает взаимодействие с интерактивными элементами внутри миниатюры
- Возможная оптимизация: обернуть в `React.memo` чтобы не перерендерить все миниатюры при каждом изменении

### Риски и оптимизация

- **Производительность**: Рендер 10+ полноценных `MobilePreviewFrame` может быть тяжелым. Mitigation: `React.memo`, `isReadOnly` (без стейта интерактивности), и рендер только для развернутого урока (уже так работает)
- **Видео/iframe**: YouTube iframes внутри миниатюр будут тяжелыми. Для видео-блоков можно оставить упрощенный превью (иконка Play на черном фоне)
- **Rive-анимации**: Отключить маскот в миниатюрах через проп
