
# Мультиязычные курсы: AI-перевод + ручная правка

## Концепция

Автор создает курс на одном языке (основном), затем нажимает "Перевести" — AI переводит весь контент на выбранные языки. Переводы можно редактировать вручную. Ученик видит курс на языке браузера, с возможностью переключить вручную.

## Как это будет работать

### Для автора (редактор)
1. В настройках курса появится раздел **"Языки"** — выбор основного языка и добавление языков перевода
2. В шапке редактора появится **переключатель языка** (флажки/коды) — при переключении все поля (тексты блоков, варианты ответов, заголовки уроков) показывают контент на выбранном языке
3. Кнопка **"Перевести на все языки"** — AI переводит весь контент с основного языка
4. Изменённые блоки помечаются как "нужен повторный перевод" (если основной язык обновился)

### Для ученика (плеер)
1. При открытии курса язык определяется по браузеру (`navigator.language`)
2. На карте уроков — маленький переключатель языка (если курс доступен на нескольких)
3. Если перевод на язык ученика не найден, показывается основной язык

## Технический план

### 1. Новые таблицы в базе данных

```text
course_languages
 ├── id (uuid, PK)
 ├── course_id (uuid, FK -> courses)
 ├── language_code (text) — 'ru', 'en', 'es', 'de', 'fr'
 ├── is_primary (boolean) — основной язык
 └── created_at

slide_translations
 ├── id (uuid, PK)
 ├── slide_id (uuid, FK -> slides)
 ├── language_code (text)
 ├── content (text) — переведённый текст блока
 ├── options (jsonb) — переведённые варианты ответов
 ├── explanation (text)
 ├── explanation_correct (text)
 ├── explanation_partial (text)
 ├── hints (jsonb)
 ├── blank_word (text)
 ├── matching_pairs (jsonb)
 ├── ordering_items (jsonb)
 ├── sub_blocks (jsonb) — переведённые sub-блоки
 ├── is_stale (boolean) — помечается true, когда основной язык обновился
 ├── created_at
 └── updated_at

lesson_translations
 ├── id (uuid, PK)
 ├── lesson_id (uuid, FK -> lessons)
 ├── language_code (text)
 ├── title (text)
 ├── description (text)
 └── updated_at

published_slide_translations (аналогично slide_translations, но для published_slides)
published_lesson_translations (аналогично lesson_translations, но для published_lessons)
```

RLS-политики: автор может CRUD свои переводы, публичные переводы доступны всем для чтения (аналогично текущим политикам).

### 2. Edge-функция `translate-course`

- Принимает `course_id`, `target_languages[]`, `source_language`
- Загружает все уроки и слайды на основном языке
- Формирует batch-запрос к Gemini (gemini-3-flash-preview) с контекстом курса
- Переводит: заголовки уроков, контент блоков, варианты ответов, объяснения, хинты, matching pairs, ordering items, текст в sub-блоках
- НЕ переводит: медиа (картинки, видео, аудио), дизайн-настройки, цвета
- Сохраняет результаты в `slide_translations` и `lesson_translations`
- Возвращает прогресс (можно реализовать через polling или SSE)

### 3. Изменения в редакторе (`Editor.tsx`)

- Новый стейт `currentEditLanguage` — какой язык сейчас редактируется
- Переключатель языков в `EditorHeader` — компактные кнопки с кодами языков (RU, EN, ES...)
- Когда `currentEditLanguage !== primaryLanguage`: данные блоков подтягиваются из `slide_translations`, при редактировании обновляется перевод
- Индикатор "устаревший перевод" на блоках, где `is_stale = true`

### 4. Изменения в плеере (`CoursePlayerV2`, `useCourseData`)

- `useCourseData` получает параметр `language`
- При загрузке опубликованного курса: сначала загружаются `published_slide_translations` для нужного языка
- Если перевод есть — подменяет текстовые поля в слайдах
- Если нет — показывает основной язык
- `PublicCourse.tsx` определяет язык по `navigator.language` и query-параметру `?lang=en`

### 5. Изменения в публикации (`usePublishing`)

- При публикации также копируются переводы в `published_slide_translations` и `published_lesson_translations`

### 6. UI настройки языков

- В `CourseSettings.tsx` или отдельной вкладке в `PublishDialog`: выбор основного языка и добавление языков перевода
- Кнопка "Перевести" с индикатором прогресса
- Статус по каждому языку: "Переведено", "Частично", "Устарело"

## Порядок реализации

1. Создать таблицы + RLS-политики
2. Edge-функция перевода
3. UI настройки языков в редакторе
4. Переключатель языков в редакторе + загрузка/сохранение переводов
5. Пометка "устаревших" переводов при изменении основного языка
6. Интеграция в плеер (авто-определение + переключатель)
7. Интеграция в публикацию (копирование переводов)

## Что НЕ входит в первую версию

- Перевод аудио/видео (только текстовый контент)
- Автоматический повторный перевод при изменении основного — только пометка "устарело"
- Перевод UI плеера (кнопки "Далее", "Проверить" и т.д.) — это отдельная задача
