

# Доработка курса через свободный чат (Course Refinement)

## Проблема

В idle-режиме (без активных инструментов) сайдбар отправляет запрос в `subblock-ai` -- функцию для редактирования отдельных блоков. Она не знает о структуре курса и не может добавить урок, убрать все квизы и т.д.

## Разделение ответственности

| Действие | Как активируется | Edge-функция |
|---|---|---|
| Создать курс с нуля | Кнопка "Создать курс" | `generate-course` |
| Редактировать один блок | Кнопка "Ред. блок" + выбор блока + промпт в чате | `subblock-ai` |
| Доработать весь курс | Свободный чат (без активных инструментов) | `refine-course` (новая) |

## Техническая реализация

### 1. Новая Edge-функция: `supabase/functions/refine-course/index.ts`

- Принимает `{ message, lessons, designSystem? }` -- сжатое представление всех уроков курса + промпт пользователя
- Системный промпт объясняет ИИ структуру курса и доступные действия:
  - Добавлять/удалять уроки
  - Менять контент блоков массово (убрать все квизы, упростить тексты и т.д.)
  - Менять порядок уроков
- Возвращает `{ message, lessons }` -- полный обновленный массив уроков
- Использует `GEMINI_API_KEY` (уже настроен), модель `gemini-3-flash-preview`

### 2. Новый хук: `src/hooks/useRefineCourse.ts`

- Функция `refineCourse(prompt, currentLessons, designSystem)`
- Конвертирует `Lesson[]` в сжатый JSON, вызывает Edge-функцию, парсит ответ обратно в `Lesson[]`

### 3. Изменения в `EditorAISidebar.tsx`

Новые пропсы:
- `allLessons: Lesson[]` -- все уроки курса
- `onRefineCourse: (lessons: Lesson[]) => void` -- колбэк для применения результата

Логика `handleFreeChat` (idle-режим, инструменты не активны):
- Вызывает `refine-course` с полным курсом
- Показывает ответ ассистента в чате
- Применяет обновленные уроки через `onRefineCourse`

### 4. Изменения в `Editor.tsx`

- Передать `allLessons={course.lessons}` в сайдбар
- Добавить `onRefineCourse` колбэк, который заменяет `course.lessons` на результат

### 5. Обновить `supabase/config.toml`

- Добавить `[functions.refine-course]` с `verify_jwt = false`

## Что НЕ меняется

- `generate-course` -- создание нового курса
- `subblock-ai` -- точечное редактирование блока (активируется кнопкой "Ред. блок")
- `useAIGeneration.tsx` -- состояние генерации
- `useGenerateCourse.ts` -- логика генерации

## UX-сценарий

1. Пользователь создал курс (3 урока), убрал инструменты
2. Пишет в чате "добавь 4й урок"
3. Появляется сообщение + индикатор загрузки
4. `refine-course` получает 3 урока + промпт, возвращает 4 урока
5. Ассистент: "Добавил урок 4: Пляжи Тайланда"
6. Курс обновляется
